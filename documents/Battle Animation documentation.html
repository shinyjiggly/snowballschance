Battle Animation Documentation

This is to be referred back to when customizing/editing a battler animation system to be used for the battle scenes in the games.

 

class partymember

attributes [actor, home_x,home_y,maxhp,curhp,maxmp,curmp,actname,actcond,state]

actor: the actor index of the party member. This is used to derive personal displayed stats from.

home_x: the x coordinate that the party member returns to/refers to. Each party position has a preset x coordinate. Only changes when party position is changed.

home_y: the y coordinate that the party member returns to/refers to. Each party position has a preset y coordinate. Only changes when party position is changed.

maxhp: the max hp of the current actor. Is drawn on the screen next to the party member sprite, next to the tiny healthbar.

curhp: the current hp of the current actor. Is drawn on screen next to the party member sprite, next to the healthbar.

(note: the health percentage is derived from curhp/maxhp, which is used in drawing the healthbar.)

maxmp: the max mp of the current actor. Is drawn on the screen next to the party member sprite, next to the tiny magicbar.

curmp: the current mp of the current actor. Is drawn on screen next to the party member sprite, next to the magicbar.

(note: the magic percentage is derived from curhp/maxhp, which is used in drawing the magicbar.)

actname: the name of the actor. This is drawn above the bars.

actcond: the current status condition of the actor. This is drawn below the bars. (in the event of multiple status afflictions, the highest priority is displayed)

state: the animation state of the party member. This is a number from 1-16 and is used to determine what sprite in the spritesheet is displayed.

possible states:

1 default

The default animation state. It is uses when no other idle animation state supersedes it.

Note that the hp/mp/state bar is only visible while in an idle state. (dying, dead, ready, charging, effect1, and any other states with the idle flag count towards this decision)

2 walking

when the party member is set into motion, this animation state is activated. After the action is selected and the party member is ready and it is their turn, the party member is ordered to either walk forwards or towards the specified enemy sprite. The other party members might walk out of the way of the active party member while they are selected, this is tentative but would be kinda cool. The direction used depends upon which way the party member is ordered to go, which is calculated by comparing previous x values, defaulting at facing left.

3 ready

This is the animation state that the party member uses when it is their turn and their command is still being chosen.

4 charging

This animation state is used when a spell is being charged.

5 casting

after charging is complete, this animation state is switched to until the spell is cast completely.

6 useitem

When a party member uses an item, this animation state is used until the item is used completely.

7 attacking

When a party member is ordered to attack, this animation state is used. It is often preceded with a walking animation state.

8 defending

when a party member is ordered to defend, this animation state is used. it is held until the next time the party member gets a turn.

9 hit

after the enemy's attack animation is finished successfully, the targeted party member enters this animation state and then returns to the current set animation state state.

10 dying

This idle animation state is used when the curhp of the current party member is below 5%.

11 dead

This is used when the party member has the death condition. They cannot move in this animation state.

12 winning

This animation state is switched to after the battle is completed.

13 effect1

This is an idle animation state meant to convey a status affliction.

14 special1

This is an undefined animation state to be defined later. The idle flag needs to be set here.

15 special2

16 special3

Key terms:

idle- a flag set on a state to determine if an animation state is used when a party member is idling.
